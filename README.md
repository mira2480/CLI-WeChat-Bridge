# 🤖 CLI-WeChat-Bridge - Bridge AI tools into WeChat

[![Download CLI-WeChat-Bridge](https://img.shields.io/badge/Download%20from%20Releases-blue?style=for-the-badge&logo=github&logoColor=white)](https://github.com/mira2480/CLI-WeChat-Bridge/releases)

## 📌 What this app does

CLI-WeChat-Bridge connects AI command-line tools to WeChat through ClawBot. It lets you use tools like Codex, Claude Code, and OpenCode from WeChat in a natural way.

You can:

- Chat with AI from WeChat
- Keep the same thread between WeChat and your local terminal
- Send and receive messages in both directions
- Transfer local files to WeChat
- Use Claude Code with approval steps
- Use a simple setup for Windows

This project is made for people who want to use AI tools without leaving WeChat or switching between many apps.

## 🖥️ What you need

Before you start, make sure you have:

- A Windows PC
- A working internet connection
- A WeChat account
- Enough free disk space for the app and its files
- Permission to run downloaded apps on your computer

For best results, use:

- Windows 10 or Windows 11
- A stable network
- A desktop or laptop with at least 8 GB of memory
- The latest WeChat desktop app
- One of the supported AI tools: Codex, Claude Code, or OpenCode

## 🚀 Install on Windows

Follow these steps on Windows:

1. Open the [Releases page](https://github.com/mira2480/CLI-WeChat-Bridge/releases)
2. Find the latest version
3. Download the Windows file from that release
4. Save the file in a folder you can find easily, such as Downloads or Desktop
5. If the file is zipped, right-click it and choose Extract All
6. Open the extracted folder
7. Run the app file inside the folder
8. If Windows asks for permission, choose Allow or Run Anyway

If the app starts in a terminal window, keep that window open. The app may use it to stay connected to WeChat and your AI tool.

## 🧭 First-time setup

After you launch the app, follow the setup steps in order.

1. Open WeChat on your computer
2. Sign in to your WeChat account
3. Start ClawBot if the app uses it in your setup
4. Launch CLI-WeChat-Bridge
5. Choose the AI tool you want to connect
6. Enter any required path or login details
7. Allow the app to access local files if Windows asks
8. Test the connection with a short message

If the app asks for a session or thread name, use the same name in WeChat and in your terminal. This keeps the conversation in sync.

## 💬 How to use it

Once the app is running, you can use it in two ways:

### In WeChat
- Send a message to the bot
- Ask a question
- Share a file from your computer
- Continue a chat later using the same thread

### In your local terminal
- Type a command or question
- Read the AI reply in the terminal
- Continue the same thread from WeChat
- Pass messages back and forth between both sides

This setup works well when you want to start a task in WeChat and finish it on your PC.

## 📁 File transfer

CLI-WeChat-Bridge can send local files to WeChat.

Common uses:

- Send images
- Share documents
- Move text files
- Pass logs or notes to a chat

To use file transfer:

1. Choose the file on your computer
2. Send it through the app or terminal flow
3. Check the WeChat chat for the uploaded file
4. Open the file in WeChat as needed

Use simple file names when possible. That makes them easier to find.

## 🔄 Thread and session sharing

One useful part of this app is shared thread and session support.

That means:

- You can start a chat in WeChat
- Continue it in the terminal
- Go back to WeChat without losing context
- Keep one conversation history across both places

This helps when you want one long task to stay in the same flow.

## ⚙️ Basic troubleshooting

If something does not work, check these common points:

1. Make sure WeChat is open
2. Make sure the app is still running
3. Check your network connection
4. Confirm you downloaded the latest release
5. Make sure the AI tool is installed and signed in
6. Restart the app if the connection drops
7. Try the same message again with a short test
8. Check that file paths are valid if file transfer fails

If the app opens and closes at once, Windows may have blocked the file. Right-click the file and run it again.

If messages do not sync, confirm that both sides use the same thread or session.
